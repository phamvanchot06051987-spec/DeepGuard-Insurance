🛡️ DeepGuard 智能保险管家中台 - 完整部署指南这是一个基于 DeepSeek 大模型和 Streamlit 框架构建的智能保险资产管理平台。本项目包含了：图像处理（OCR 前置）、保单结构化解析、家族全景资产看板、车险理赔智能预判、以及基于长文本的合同条款智能问答功能。为了让系统顺利跑起来，请严格按照以下指南操作。📁 零、确认目录结构在开始部署之前，请确保你的项目文件夹内完整包含以下 4 个文件，且文件名完全一致：保单优化系统.py (主程序代码文件，包含所有前端UI和后端逻辑)requirements.txt (Python 依赖清单，告诉服务器需要安装哪些库)Dockerfile (容器化打包配置文件，用于私有化部署)README.md (本说明文档)💻 一、本地运行测试（上线前必做）在将项目传到云端之前，建议先在自己的电脑上跑通，确认代码无误。1. 安装 Python 环境请确保你的电脑上安装了 Python 3.9 或更高版本（推荐 Python 3.10）。2. 安装依赖包打开终端（命令行 / Terminal），进入到项目所在的文件夹，运行以下命令安装必备的运行库：pip install -r requirements.txt
3. 启动系统依赖安装完成后，在同一个终端窗口运行：streamlit run 保单优化系统.py
运行后，终端会弹出一个 Local URL（通常是 http://localhost:8501），你的默认浏览器会自动打开这个网址。这就说明本地运行成功了！🚀 二、部署方案一：零成本云端极速部署（强烈推荐）如果你没有自己的云服务器，想免费获得一个公网网址（比如 https://your-insurance-app.streamlit.app），并且能随时在手机上发给客户或自己使用，请使用此方案。第一步：将代码上传至 GitHub注册并登录 GitHub 账号。点击右上角的 + 号，选择 New repository（新建仓库）。给仓库起个名字（例如 DeepGuard-Insurance），设为 Public（公开）或 Private（私有）均可，点击 Create。在仓库页面，点击 "uploading an existing file"，将准备好的 4 个文件全部拖拽上传，然后点击 Commit changes 保存。第二步：在 Streamlit 部署打开并登录 Streamlit Community Cloud（可以直接使用刚才的 GitHub 账号授权登录）。点击右上角的 "New app" 按钮。授权 Streamlit 访问你的 GitHub 仓库。在部署页面填写信息：Repository: 选择你刚才创建的仓库（如 your_name/DeepGuard-Insurance）。Branch: 保持默认的 main。Main file path: 填写主程序文件名 保单优化系统.py。点击右下角的 "Deploy!"。等待约 1-3 分钟，服务器会自动安装 requirements.txt 里的库，完成后页面会自动刷新出你的应用！🐳 三、部署方案二：私有化服务器部署 (Docker)如果你有一台云服务器（如阿里云、腾讯云、华为云），希望数据完全留在自己的服务器上，并配置自己的专属域名，请使用 Docker 部署方案。第一步：服务器环境准备通过 SSH 登录到你的云服务器（操作系统推荐 Ubuntu 20.04 或 CentOS）。确保服务器上已经安装了 Docker。如果没有，请先运行命令安装 Docker：curl -fsSL [https://get.docker.com](https://get.docker.com) | bash -s docker
使用 FTP 工具（如 FileZilla、Xftp）或 Git，将项目文件夹上传到服务器的某个目录下（例如 /root/insurance_project/）。第二步：构建镜像 (Build)在服务器终端中，进入你上传的项目目录，执行以下命令把代码打包成镜像（注意命令最后面有一个英文句号 .）：cd /root/insurance_project/
docker build -t deepguard-ai .
此过程会自动拉取 Python 环境并安装依赖，需要几分钟时间，请耐心等待。第三步：运行容器并挂载数据 (Run)为了防止容器重启后，你辛苦录入的“家族保单数据”丢失，我们必须把容器内的数据库文件挂载到服务器的实体硬盘上。执行以下命令：docker run -d \
  --name my-insurance-app \
  -p 80:8501 \
  -v $(pwd)/insurance_data:/app \
  deepguard-ai
命令详细解释：-d: 让程序在后台静默运行。--name: 给这个运行的容器起个名字叫 my-insurance-app。-p 80:8501: 端口映射。把服务器的外网 80 端口（默认HTTP端口）映射到容器内部的 8501 端口。这样你直接在浏览器输入 IP 就能访问，不用加 :8501。-v $(pwd)/insurance_data:/app: 数据持久化。在当前目录下创建一个 insurance_data 文件夹，并与容器内部目录打通。这样产生的 insurance_ultimate.db 数据库就会永久保存在你的服务器硬盘上。第四步：访问测试打开手机或电脑的浏览器，直接输入你这台服务器的 公网 IP 地址（例如 http://123.45.67.89），即可看到系统界面！💡 四、常见问题与避坑指南 (FAQ)Q1: 云端部署后，上传图片解析报错 libGL.so.1: cannot open shared object file 怎么办？A1: 这是因为云端没有显示器驱动。请务必检查你的 requirements.txt 文件，里面必须写的是 opencv-python-headless，而绝对不能写 opencv-python。Q2: 为什么我的 API Key 填进去之后，一刷新页面就没了？A2: 系统已在代码中启用了 session_state 缓存机制。请在页面的【⚙️ 设置】标签页中输入 API Key 并点击保存配置，只要你不关闭当前的浏览器标签页，Key 就一直有效。Q3: 车险理赔和条款问答功能无法使用？A3: 确保你填入的 DeepSeek API Key 账户内有足够余额。DeepSeek API 注册后通常需要充值（几块钱即可）才能正常调用大模型接口。
